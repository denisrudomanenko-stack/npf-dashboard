## Конфигурация LLM

### Мульти-провайдерная стратегия

```
                    ┌─────────────────┐
                    │   RAG Service   │
                    └─────────────────┘
                           │
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
     ┌─────────────┐┌─────────────┐┌─────────────┐
     │   Ollama    ││   Ollama    ││  Anthropic  │
     │   (Chat)    ││ (Embedding) ││  (Fallback) │
     │qwen2.5:7b   ││nomic-embed  ││claude-sonnet│
     └─────────────┘└─────────────┘└─────────────┘
           │              │              │
           │   Primary    │   Primary    │  Fallback
           └──────────────┴──────────────┘
```

### Runtime переключение

Конфигурация LLM хранится в БД (таблица `llm_config`) и загружается при старте приложения. Изменения применяются без перезапуска сервера.

## Безопасность

- **CORS** — настроен для localhost:3000, 3004, 5173
- **Валидация файлов** — проверка типов и размеров при загрузке
- **Дедупликация** — SHA-256 хеш для предотвращения дублей
- **Soft delete** — документы не удаляются физически
- **JWT готовность** — структура для аутентификации подготовлена

### Рекомендации для Production

1. **БД**: Миграция на PostgreSQL
2. **Векторное хранилище**: Отдельный сервер ChromaDB или Milvus
3. **LLM**: Выделенный GPU-сервер для Ollama
4. **Кэширование**: Redis для сессий и кэша запросов
5. **Мониторинг**: Prometheus + Grafana
